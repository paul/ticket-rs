# Add priority display and sorting to tree command

The tree command should expose ticket priority (P0, P1, etc.) in its output. Priority should be visually indicated with coloring: P0 items should use a highlight/attention color, while P4 items should be dimmed. Items within the tree should be ordered by priority (highest first).

## Acceptance Criteria

Tree output shows priority label (P0-P4) for each ticket. P0 tickets are rendered with a high-visibility color (e.g. red or bold). P4 tickets are rendered with dim/muted styling. Sibling items within each level of the tree are sorted by priority (P0 first, P4 last). Existing tree features (depth limit, --all, dep display, tags, truncation) continue to work correctly.
