# Extract line format helpers and unify list/dep output with tree

The tree, list, and dep commands each have their own ticket-line rendering. tree.rs has the most polished format (colored id/priority/status, visible deps colored by their status, dimmed tags, terminal-width truncation). list.rs has four separate format functions with no color or truncation. dep.rs has its own plain-text tree renderer.

Create a new src/format.rs module containing the shared helpers extracted from tree.rs: display_width(), status_label(), priority_label(), dep_id_label(), and build_line(). Make build_line() work for flat output too (empty prefix and connector produce a plain ticket line).

Update tree.rs to import from format instead of defining these locally.

Update list.rs: replace format_line, format_ready_line, format_blocked_line, and format_closed_line with calls to build_line(). Add TTY detection so flat list output also gets terminal-width truncation. Update the BDD feature tests in ticket_listing.feature to match the new output format.

Update dep.rs: replace its custom node rendering with calls to build_line().
