# Implement edit command

Create src/commands/edit.rs. Implement edit <id>. Resolve partial ID to full file path. If stdout is a TTY, open the file in $EDITOR (fall back to vi if EDITOR not set). If not a TTY (non-interactive/agent context), print the file path to stdout instead. Use std::process::Command to launch the editor and wait for it to exit.

## BDD Integration Tests

```bash
TICKET_SCRIPT=./target/debug/ticket behave features/ticket_edit.feature
```

Scenarios cover: non-TTY mode prints the file path (not launching an editor), non-existent ticket error, and partial ID resolution. The BDD suite always runs non-TTY (`stdin=subprocess.DEVNULL`), so editor launch is never exercised — it's tested manually or via a separate unit test that mocks `$EDITOR`.
