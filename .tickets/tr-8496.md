# Fall back to show when bare ticket ID is passed as command

When a user runs `tk abc-1234` with a valid ticket ID (not a known subcommand or plugin), fall back to `tk show abc-1234`. If the argument doesn't resolve to a ticket, show the help text instead of the current 'unknown command' error.

## Implementation

Modify the `Commands::External` dispatch arm in `main.rs`. When `plugin::find_plugin` returns `None`:

1. If there is exactly one argument and it resolves via `TicketStore::find(None)` + `store.resolve_id()`, call `commands::show()` with that ID.
2. Otherwise, call `print_help_with_plugins()` and exit.

This keeps the change minimal — no new CLI definitions, no pre-parse hacks — and leverages the existing `External` catch-all that clap already provides.
