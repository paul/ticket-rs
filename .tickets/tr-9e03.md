# Extract sort helpers into Ticket and Status

The sort comparison logic is duplicated in list.rs (priority->id, 3 times) and tree.rs (status_priority() free function, used in multiple sort closures). There is no shared sort key on Ticket or Status, so each command re-implements ordering independently.

Add Status::sort_key() -> u8 (in_progress=0, open=1, closed=2) and Ticket::sort_cmp(&self, other: &Ticket) -> Ordering (status -> priority -> created -> id) to ticket.rs.

Update list.rs to use Ticket::sort_cmp() instead of its three priority->id sort closures. This is a minor behavior change: ls, ready, and blocked will now group by status (in_progress first) and use created as a tiebreaker before id, matching tree's ordering.

Update tree.rs to use Status::sort_key() instead of the local status_priority() free function.
