# Replace static completions with clap_complete dynamic completions

Replace the hand-written completions/_tk zsh script with clap_complete's dynamic completion system (unstable-dynamic feature). When the user presses Tab, the shell invokes the tk/ticket binary itself via the COMPLETE env var to resolve completions in real-time, including ticket IDs from the .tickets/ directory.

Changes:
- Add clap_complete dependency with unstable-dynamic feature
- Create src/complete.rs with a custom ValueCompleter that scans .tickets/ for ticket IDs
- Annotate id arguments in cli.rs with ArgValueCompleter
- Wire up CompleteEnv in main.rs (first line of main, before any output)
- Delete completions/_tk and the completions/ directory
- Update Justfile: replace static completions copy with an interactive prompt asking to install shell completions, remove comp_dir variable, update uninstall recipe
