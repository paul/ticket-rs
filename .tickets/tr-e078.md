# Add --help handling to ticket subcommands

Agents tried `<subcommand> --help` 24 times across session history and got confusing errors each time:

- `ticket update --help` â†’ `Error: ticket '--help' not found` (clap passes it through to the `update` command which treats it as a ticket ID)
- `ticket query --help` â†’ the filter string `--help` is passed to jq, which fails with a compile error
- `ticket create --help` â†’ clap rejects the unknown flag
- `ticket list --help` (via plugin) â†’ silently returns all tickets

The root issue is that `ticket <subcommand> --help` is not being handled. For the built-in clap commands this is automatically solved. The affected commands are the external plugins (`query`, `list`/`ls`) and the `update` command which interprets its first positional arg as a ticket ID before seeing `--help`.

## Design

**Built-in clap commands** already get `--help` for free from clap â€” `ticket create --help`, `ticket status --help`, `ticket add-note --help`, etc. all work correctly in the Rust binary. No changes needed for built-ins.

**`ticket update` plugin** (`~/.local/bin/ticket-update` / the plugin discovered on PATH):

The plugin's arg parser takes `$1` as the ticket ID unconditionally. Add an early-exit check before argument parsing:

```bash
if [[ "${1:-}" == '--help' || "${1:-}" == '-h' ]]; then
    # reuse the existing usage block that prints when $# -lt 1
    print_usage
    exit 0
fi
```

However, since this is a Rust project, the `update` command is a built-in (see `src/commands/update.rs` and `Commands::Update` in `src/cli.rs`). The `id` field is a required positional argument, so clap sees `ticket update --help` before trying to parse `id` and correctly prints help. The bash `ticket-update` plugin in PATH is a legacy artifact; if it is still shadowing the built-in, it should be removed or updated.

**`ticket query` plugin** (`src/commands/query.rs` + `Commands::Query` in `src/cli.rs`):

`Query` is a built-in. `ticket query --help` already works via clap. The jq error only occurs when using the old bash `ticket-query` plugin. If the bash plugin is still on PATH and shadowing the built-in, remove it or add a `--help` intercept to it.

**`ticket list`/`ls`**:

`Ls` is a built-in with `#[command(alias = "list")]`. `ticket list --help` and `ticket ls --help` already work via clap. Same resolution as above â€” ensure the bash `ticket-list` plugin is not shadowing the built-in.

**Summary of actual work:**

1. Verify that the bash plugins (`ticket-update`, `ticket-query`, `ticket-list`) in PATH are not shadowing the Rust built-ins.
2. If they are still needed (e.g. during a transition period), add `--help`/`-h` intercepts to each.
3. Confirm that `ticket <subcommand> --help` works for all built-in commands.

## Acceptance Criteria

- `ticket create --help` prints clap-generated usage and exits 0
- `ticket update --help` prints clap-generated usage and exits 0
- `ticket query --help` prints clap-generated usage and exits 0
- `ticket list --help` prints clap-generated usage and exits 0
- `ticket status --help` prints clap-generated usage and exits 0
- `ticket add-note --help` prints clap-generated usage and exits 0
- All `--help` calls exit 0
