# Add --help handling to tk subcommands and plugins

Agents tried `<subcommand> --help` 24 times across session history and got confusing non-help output each time:
- `tk update --help` â†’ `Error: ticket '--help' not found` (treated as ticket ID)
- `tk query --help` â†’ jq compile error (passed to jq as a filter)
- `tk create --help` â†’ `Unknown option: --help`
- `tk list --help` â†’ returns all tickets unfiltered (silently ignores flag)

Also, `tk help <subcommand>` just dumps the full help page regardless of argument â€” subcommand-specific help is unavailable.

Files affected: `/home/rando/.local/bin/tk`, `/home/rando/.local/bin/ticket-update`, `/home/rando/.local/bin/ticket-list`, `/home/rando/.local/bin/ticket-query`.

## Design

**Built-in commands (`/home/rando/.local/bin/tk`):**
Each `cmd_*` function should check its first argument and short-circuit with usage if it is `--help` or `-h`. The usage text can be a brief one-liner matching what `tk help` already shows, e.g.:
```bash
cmd_create() {
    [[ "${1:-}" == '--help' || "${1:-}" == '-h' ]] && { echo 'Usage: tk create [title] [options]'; echo '  --title, -t, --type, -d/--description, --body, --design, --acceptance, -p, -a, --parent, --tags, --external-ref'; return 0; }
    ...
```
Apply the same pattern to: `cmd_create`, `cmd_status`, `cmd_show`, `cmd_add_note`, `cmd_dep`, `cmd_undep`, `cmd_link`, `cmd_unlink`, `cmd_ready`, `cmd_blocked`, `cmd_closed`, `cmd_start`, `cmd_close`, `cmd_reopen`.

**`ticket-update`:**
Already has usage output â€” add an early check:
```bash
if [[ "${1:-}" == '--help' || "${1:-}" == '-h' ]]; then
    # print existing usage block
    exit 0
fi
```

**`ticket-query`:**
Intercept `--help`/`-h` before the filter is passed to jq:
```bash
if [[ "${1:-}" == '--help' || "${1:-}" == '-h' ]]; then
    echo 'Usage: tk query [jq-filter]'
    echo 'Outputs tickets as JSONL (one JSON object per line). Optionally filter with a jq select() expression.'
    echo 'Example: tk query '\''select(.status == "open" and (.tags | index("spike")))'\'''
    exit 0
fi
```

**`ticket-list`:**
Add `--help`/`-h` to the arg parser:
```bash
--help|-h) echo 'Usage: tk list [-T tag] [--tag=tag] [--tags tag] [--status=status] [-a assignee]'; exit 0 ;;
```

**`tk help <subcommand>` (`cmd_help`):**
Make `cmd_help` accept an optional argument; if provided, delegate to the relevant `cmd_*` function with `--help`. For plugin subcommands, dispatch to the plugin with `--help`.

## Acceptance Criteria

- `tk create --help` prints usage for create and exits 0
- `tk update --help` prints usage for update and exits 0
- `tk query --help` prints usage for query (including JSONL note) and exits 0
- `tk list --help` prints usage for list and exits 0
- `tk status --help` prints usage for status and exits 0
- `tk add-note --help` prints usage for add-note and exits 0
- `tk help create` shows create-specific help
- `tk help update` shows update-specific help
- All --help calls exit 0 (not 1)
